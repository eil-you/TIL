# 자바 기초: 자바(Java)란 무엇인가?

자바를 배우기 전, 자바가 왜 만들어졌고 어떤 특징을 가지고 있는지 이해하는 것은 매우 중요합니다.

---

## 1. 자바(Java)의 정의
자바는 1995년 선 마이크로시스템즈(Sun Microsystems, 현재는 Oracle)에서 발표한 **객체 지향 프로그래밍 언어(OOP)**입니다. 

### "Write Once, Run Anywhere (WORA)"
자바의 가장 큰 슬로건입니다. 한 번 작성한 코드는 운영체제(Windows, Linux, macOS 등)에 상관없이 어디서든 실행될 수 있다는 의미입니다.

---

## 2. 자바의 주요 특징

### 1) 플랫폼 독립성
보통의 언어는 특정 OS에 맞춰 컴파일되지만, 자바는 **JVM(Java Virtual Machine)**이라는 가상 머신 위에서 돌아갑니다. OS에 맞는 JVM만 설치되어 있다면 동일한 자바 실행 파일(`.class`)을 그대로 실행할 수 있습니다.

### 2) 객체 지향 언어 (Object-Oriented Programming, OOP)
세상을 객체(Object)라는 단위로 나누고, 이 객체들의 상호작용으로 프로그램을 구성합니다. 코드의 재사용성과 유지보수성이 매우 높습니다.

### 3) 자동 메모리 관리 (Garbage Collection)
개발자가 직접 메모리를 해제할 필요가 없습니다. 쓰지 않는 메모리는 **가비지 컬렉터(GC)**가 알아서 수거해 가므로, 메모리 누수 걱정을 덜어줍니다.

### 4) 풍부한 오픈소스와 생태계
전 세계적으로 가장 많이 쓰이는 언어 중 하나로, 스프링(Spring) 프레임워크 등을 포함해 방대한 라이브러리와 커뮤니티가 형성되어 있습니다.

---

## 3. 자바 프로그램의 동작 원리

개발자가 코드를 작성하고 실행하기까지의 과정은 다음과 같습니다.

1.  **코드 작성**: `Hello.java` 파일 작성
2.  **컴파일**: 자바 컴파일러(`javac`)가 소스 코드를 **바이트코드(`Hello.class`)**로 변환
    *   *바이트코드는 인간과 컴퓨터 사이의 중간 단계 언어입니다.*
3.  **실행**: **JVM**이 바이트코드를 읽어서 해당 OS가 이해할 수 있는 기계어로 해석하고 실행

---

## 4. JDK, JRE, JVM 구분하기

공부를 시작할 때 가장 헷갈리는 용어 3가지입니다.

*   **JVM (Java Virtual Machine)**: 자바 바이트코드를 실행하는 핵심 엔진.
*   **JRE (Java Runtime Environment)**: JVM + 자바 표준 라이브러리. 자바 프로그램을 **실행**하기 위해 필요한 최소 환경.
*   **JDK (Java Development Kit)**: JRE + 개발 도구(컴파일러 등). 자바를 **개발**하려면 반드시 JDK를 설치해야 합니다.
    *   *최근 버전에서는 JDK만 설치하면 JRE와 JVM이 모두 포함되어 있습니다.*

---

## 5. 핵심 포인트 (Q&A)

### Q. 자바는 속도가 느리지 않나요?
A. 과거에는 가상 머신을 거치기 때문에 느리다는 평가가 있었으나, 현대의 JVM은 **JIT(Just-In-Time) 컴파일러**와 최적화 기술을 통해 C++에 육박하는 높은 성능을 보여줍니다.

### Q. 왜 자바를 배워야 하나요?
A. 대형 엔터프라이즈 시스템(은행, 공공기관 등), 안드로이드 앱 개발, 빅데이터 처리 등 매우 다양한 분야에서 표준처럼 쓰이고 있어 취업 및 실무 활용도가 가장 높기 때문입니다.

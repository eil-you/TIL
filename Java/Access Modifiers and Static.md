# Java: 접근 제어자(Access Modifier)와 static

자바에서 클래스, 변수, 메서드를 정의할 때 "누가 이 데이터에 접근할 수 있는가"와 "이 데이터가 어디에 속하는가"를 결정하는 것은 매우 중요합니다.

---

## 1. 접근 제어자 (Access Modifier)

접근 제어자는 객체 지향의 핵심 원칙인 **캡슐화(Encapsulation)**를 구현하기 위해 사용됩니다. 외부로부터 데이터를 보호하고 불필요한 노출을 막는 역할을 합니다.

| 제어자 | 같은 클래스 | 같은 패키지 | 자식 클래스 (다른 패키지) | 전체 (전부) |
| :--- | :---: | :---: | :---: | :---: |
| **public** | O | O | O | O |
| **protected** | O | O | O | X |
| **default** (비워둠) | O | O | X | X |
| **private** | O | X | X | X |

### 상세 설명
1.  **public**: 어떤 클래스에서도 접근이 가능합니다. 주로 API나 공용 메서드에 사용합니다.
2.  **protected**: 같은 패키지 내에서는 자유롭게 접근 가능하며, 다른 패키지에 있더라도 **상속받은 자식 클래스**라면 접근을 허용합니다.
3.  **default (package-private)**: 제어자를 적지 않았을 때의 기본값입니다. **같은 패키지** 내에서만 접근이 가능합니다.
4.  **private**: **해당 클래스 내부**에서만 접근이 가능합니다. 외부에서 직접 데이터를 수정하지 못하도록 막을 때 필수적입니다. (Getter/Setter 사용의 이유)

---

## 2. static (정적 제어자)

`static`은 접근 제어자가 아니라 **"메모리에 어떻게 올라가는가"**를 결정하는 예약어입니다.

### 핵심 개념: 인스턴스 멤버 vs 정적(static) 멤버
- **인스턴스 멤버**: 객체(`new`)를 생성할 때마다 각각의 객체 안에 생기는 데이터.
- **정적(static) 멤버**: 객체 생성과 상관없이 **클래스 자체에 고정**된 데이터. 프로그램 시작 시 메모리(Method Area)에 딱 로드됩니다.

### 사용 사례
1.  **static 변수 (공유 변수)**:
    - 모든 객체가 공통적으로 가져야 하는 값이 있을 때 사용합니다.
    - 예: `Card` 클래스의 '카드 너비/높이' (모든 카드는 크기가 같아야 하므로).
2.  **static 메서드 (유틸리티 메서드)**:
    - 객체의 상태(인스턴스 변수)를 쓰지 않고 값만 처리하는 경우에 사용합니다.
    - 예: `Math.abs()`, `Integer.parseInt()` 처럼 `new` 없이 바로 쓰는 메서드들.

---

## 3. 자주 하는 질문 (Q&A)

### Q. static 메서드에서 인스턴스 변수를 쓸 수 있나요?
A. **안 됩니다.** `static` 메서드는 객체가 생성되기도 전에 이미 메모리에 올라와 있기 때문에, 언제 생성될지 모르는 인스턴스 변수의 존재를 알 수 없습니다. 반대로 인스턴스 메서드에서는 `static` 변수를 마음껏 쓸 수 있습니다.

### Q. protected와 default의 차이가 헷갈려요.
A. 핵심은 **'다른 패키지의 상속 관계'**입니다. `default`는 패키지가 다르면 상속을 받아도 못 쓰지만, `protected`는 패키지가 달라도 자식 클래스라면 접근을 허락해 줍니다.

---

### 💡 요약
- **public**: 모두 오픈
- **protected**: 우리 패키지 + 다른 패키지여도 내 자식이면 오픈
- **default**: 우리 패키지만 오픈
- **private**: 나만 쓸 거야
- **static**: 객체마다 만드는 게 아니라 클래스에 딱 붙여서 **공유**하는 것

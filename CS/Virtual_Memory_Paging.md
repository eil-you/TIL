# 가상 메모리와 페이징 (Virtual Memory & Paging)

컴퓨터가 실제 물리적 메모리(RAM)의 한계를 극복하고 효율적으로 관리하기 위해 사용하는 운영체제의 메모리 관리 기법입니다.

---

## 1. 가상 메모리 (Virtual Memory)
- **정의**: 프로세스 전체가 메모리에 올라오지 않더라도 실행이 가능하도록 하는 기법입니다.
- **핵심 원리**: 사용자에게는 매우 큰 메모리가 있는 것처럼 보이게(논리 주소) 하고, 실제 데이터는 필요할 때만 물리 메모리(RAM)나 디스크(Swap 영역)에 적재합니다.
- **장점**:
  - 실제 RAM 용량보다 큰 프로그램 실행 가능.
  - 여러 프로세스가 메모리를 공유하며 효율성 증대.
  - 프로세스 간 메모리 격리로 보안 및 안정성 확보.

## 2. 페이징 (Paging) 기법
- **정의**: 가상 메모리를 **고정된 크기의 블록**으로 나누어 관리하는 기법입니다.
- **용어**:
  - **페이지 (Page)**: 가상 메모리를 나눈 고정 크기 블록.
  - **프레임 (Frame)**: 물리 메모리(RAM)를 나눈 고정 크기 블록 (페이지와 크기가 같음).
- **특징**: 페이지와 프레임이 1:1로 매핑될 필요가 없으며, 물리 메모리의 불연속적인 위치에 저장될 수 있어 **외부 단편화 문제**를 해결합니다.

## 3. 페이지 테이블 (Page Table)
- 가상 주소(논리 주소)를 물리 주소로 변환하기 위해 프로세스마다 가지는 매핑 정보 테이블입니다.
- **MMU (Memory Management Unit)**: CPU가 보낸 가상 주소를 페이지 테이블을 참조하여 실제 물리 주소로 변환해주는 하드웨어 장치입니다.

---

## 4. 주요 현상

### 페이지 폴트 (Page Fault)
- CPU가 접근하려는 페이지가 현재 물리 메모리(RAM)에 없을 때 발생하는 현상입니다.
- 운영체제는 이 보조기억장치(HDD/SSD)에서 해당 페이지를 찾아 빈 프레임에 로드합니다.

### 페이지 교체 알고리즘 (Page Replacement)
- 메모리가 꽉 찼을 때 어떤 페이지를 내보내고 새로운 페이지를 들여올지 결정하는 알고리즘입니다.
  - **FIFO**: 가장 먼저 들어온 페이지 교체.
  - **LRU (Least Recently Used)**: 가장 오랫동안 사용되지 않은 페이지 교체 (가장 많이 쓰임).

### 스래싱 (Thrashing)
- 페이지 폴트가 너무 자주 발생하여 CPU가 실제 작업보다 페이지 교체작업(Swap 인/아웃)에 더 많은 시간을 쓰는 성능 저하 현상입니다. (RAM 용량이 부족할 때 주로 발생)

---

## 5. 요약
- **가상 메모리**는 "메모리를 추상화하여 크게 쓰는 것"이고,
- **페이징**은 이를 구현하기 위해 "메모리를 똑같은 크기로 쪼개서 관리하는 방식"입니다.
- 이를 통해 현대 운영체제는 안정적이고 효율적으로 멀티태스킹을 지원합니다.

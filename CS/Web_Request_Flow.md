# 주소창에 www.naver.com을 입력했을 때 일어나는 과정

브라우저 주소창에 URL을 입력하고 엔터를 누른 순간부터 화면이 렌더링되기까지의 네트워크 및 브라우저 동작 과정을 정리합니다.

---

## 🚀 5줄 요약
1. **DNS**: 도메인 이름을 IP 주소로 변환
2. **TCP**: 3-Way Handshake를 통해 서버와 연결 통로 개설
3. **HTTP**: HTML 파일 요청(Request) 및 응답(Response)
4. **Parsing**: HTML/CSS를 분석해 설계도(DOM/CSSOM) 작성
5. **Rendering**: 위치를 잡고(Layout) 색을 칠함(Paint)

### 💡 한 줄 비유
- **DNS**: 전화번호부에서 친구 이름 찾기
- **TCP**: 전화 걸어서 "여보세요? 들려?" 확인
- **HTTP**: "물건(HTML) 좀 보내줘" -> "알았어(응답)"
- **Rendering**: 조립 설명서 보고 레고 조립하기


---

## 1. 주소 해석 및 캐싱 확인 (DNS Lookup)
사용자가 입력한 도메인 이름(`www.naver.com`)을 컴퓨터가 이해할 수 있는 **IP 주소**로 변환해야 합니다.

1.  **브라우저 캐시 확인**: 이전에 방문한 적이 있다면 브라우저에 저장된 IP를 먼저 찾습니다.
2.  **OS 캐시 및 hosts 파일 확인**: OS 수준에서 저장된 기록을 확인합니다.
3.  **DNS 서버 요청**: 캐시에 없다면 통신사(ISP)의 DNS 서버로 가서 "naver.com의 IP가 뭐야?"라고 물어봅니다. (Recursive Query)

---

## 2. TCP 연결 설정 (TCP 3-Way Handshake)
IP 주소를 알아냈다면, 해당 서버와 데이터를 주고받을 통로를 만듭니다.

- **SYN**: "연결하고 싶어요." (클라이언트 -> 서버)
- **SYN-ACK**: "네, 준비됐어요. 당신도 준비됐나요?" (서버 -> 클라이언트)
- **ACK**: "네, 이제 데이터를 보낼게요." (클라이언트 -> 서버)
- *참고: HTTPS의 경우 이 단계 이후 SSL/TLS 핸드쉐이크 과정을 추가로 거쳐 암호화 통신을 설정합니다.*

---

## 3. HTTP 요청 및 응답
연결된 통로를 통해 실제 데이터를 요청합니다.

1.  **HTTP Request**: 브라우저가 서버에게 "네이버 메인 페이지(`GET /`) 좀 보내줘"라고 요청 메시지를 보냅니다.
2.  **HTTP Response**: 서버가 요청을 확인하고 상태 코드(`200 OK`)와 함께 HTML 문서를 응답으로 보냅니다.

---

## 4. 브라우저 렌더링 (화면 띄우기)
서버로부터 받은 HTML 파일을 브라우저가 해석해서 화면에 그립니다.

1.  **DOM 트리 생성**: HTML을 읽어서 문서 객체 모델(DOM) 트리를 만듭니다.
2.  **CSSOM 트리 생성**: CSS 파일을 읽어서 스타일 규칙(CSSOM) 트리를 만듭니다.
3.  **Render 트리 생성**: DOM과 CSSOM을 합쳐서 실제로 화면에 보일 노드들만 정리한 렌더 트리를 만듭니다.
4.  **Layout (Reflow)**: 각 노드가 화면의 어디에 위치할지 기하학적인 위치를 계산합니다.
5.  **Paint**: 계산된 위치에 실제 픽셀을 채워 화면을 그립니다.

---

## 5. 연결 종료 (TCP 4-Way Handshake)
모든 데이터를 다 받았다면 연결을 안전하게 끊습니다.
- 이 과정은 `FIN`과 `ACK`를 주고받으며 4단계로 이루어집니다.

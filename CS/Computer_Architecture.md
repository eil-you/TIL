# 컴퓨터 구조 기초 (CPU, RAM, Storage)

컴퓨터의 핵심 하드웨어 구성 요소와 그 역할을 정리합니다.

---

## 1. CPU (Central Processing Unit, 중앙처리장치)
컴퓨터의 **'두뇌'**에 해당하며, 명령어 해석과 데이터 연산을 담당합니다.

- **구성 요소**:
  - **ALU (산술논리연산장치)**: 덧셈, 뺄셈 같은 산술 연산과 AND, OR 같은 논리 연산을 수행합니다.
  - **제어장치 (Control Unit)**: 명령어를 읽고 해석하여 각 장치에 신호를 보냅니다.
  - **레지스터 (Registers)**: CPU 내부의 아주 빠른 임시 저장 공간입니다.
- **특징**: 처리 속도가 가장 빠르지만 저장 용량이 매우 작습니다.

## 2. RAM (Random Access Memory, 주기억장치)
컴퓨터의 **'작업대'**에 해당하며, 실행 중인 프로그램과 데이터를 일시적으로 저장합니다.

- **특징**:
  - **휘발성 (Volatile)**: 전원이 꺼지면 저장된 데이터가 모두 사라집니다.
  - **임의 접근**: 어느 위치에 있는 데이터든 똑같이 빠른 속도로 읽고 쓸 수 있습니다.
- **역할**: CPU가 처리할 데이터를 미리 불러다 놓는 공간입니다. RAM이 클수록 여러 작업을 동시에 원활하게 처리할 수 있습니다.

## 3. 보조기억장치 (SSD, HDD)
컴퓨터의 **'창고'**에 해당하며, 데이터를 영구적으로 저장합니다.

- **SSD (Solid State Drive)**: 반도체를 이용하며 속도가 매우 빠르고 소음이 없습니다.
- **HDD (Hard Disk Drive)**: 자기 디스크를 회전시켜 읽으며 가격 대비 용량이 큽니다.
- **특징**: **비휘발성 (Non-volatile)**으로 전원이 꺼져도 데이터가 보존됩니다.

---

## 4. 데이터 흐름 요약 (계층 구조)

전체적인 데이터 처리 흐름은 아래와 같은 속도/용량 관계를 가집니다.

1. **레지스터** (가장 빠름, 가장 작음)
2. **캐시 메모리 (L1, L2, L3)** (CPU와 RAM 사이의 완충)
3. **RAM** (주기억장치)
4. **SSD/HDD** (가장 느림, 가장 큼)

> **비유**:
> - **CPU**: 요리사
> - **레지스터**: 요리사 손에 든 도구
> - **캐시**: 조리대 위의 재료
> - **RAM**: 주방 안의 작업대
> - **보조기억장치**: 냉장고/창고

---

## 5. 핵심 포인트
- **컴퓨터 속도가 느려졌다면?** 보통 RAM 용량이 부족해 보조기억장치를 작업대처럼 쓰게 되어 느려지는 경우가 많습니다 (Swapping 현상).
- **CPU의 성능**은 주로 클럭 속도(GHz)와 코어(Core) 개수로 결정됩니다.

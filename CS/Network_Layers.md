# 네트워크 계층 모델 (OSI 7계층 & TCP/IP 4계층)

네트워크 통신 과정을 단계별로 표준화한 두 가지 주요 모델을 정리합니다.

---

## 1. OSI 7계층 (Open Systems Interconnection)
ISO(국제표준화기구)에서 만든 표준 모델로, 통신 과정을 7개 단계로 세분화했습니다.

| 계층 | 이름 | 역할 | 프로토콜/장비 |
| :--- | :--- | :--- | :--- |
| **7** | **응용 (Application)** | 사용자 인터페이스 제공, 이메일, 웹 브라우징 등 | HTTP, FTP, DNS, SMTP |
| **6** | **표현 (Presentation)** | 데이터 형식 변환, 암호화, 압축 | JPEG, MPEG, ASCII, SSL/TLS |
| **5** | **세션 (Session)** | 통신 세션 유지, 동기화 | SSH, RPC, WinSock |
| **4** | **전송 (Transport)** | 데이터 전송 속도 제어, 오류 검출 및 복구 | TCP, UDP |
| **3** | **네트워크 (Network)** | 가장 효율적인 경로 설정 (라우팅), IP 부여 | IP, ICMP, Router |
| **2** | **데이터 링크 (Data Link)** | 인접 장치 간 물리적 전송, 주소 결정(MAC) | Ethernet, MAC, Switch |
| **1** | **물리 (Physical)** | 전기적 신호로 변환하여 전송 | Hub, 케이블, 리피터 |

---

## 2. TCP/IP 4계층
현재 인터넷에서 실제로 사용되는 실무 위주의 모델입니다. OSI 모델의 여러 계층을 통합하여 4개 계층으로 정의합니다.

| 계층 | 이름 | 대응하는 OSI 계층 | 역할 |
| :--- | :--- | :--- | :--- |
| **4** | **응용 (Application)** | 응용, 표현, 세션 (7, 6, 5) | 사용자와 소프트웨어 간 데이터 교환 |
| **3** | **전송 (Transport)** | 전송 (4) | 프로세스 간 신뢰성 있는 전송 보장 |
| **2** | **인터넷 (Internet)** | 네트워크 (3) | 주소 기반 경로 지정 및 데이터 전송 |
| **1** | **네트워크 인터페이스** | 데이터 링크, 물리 (2, 1) | 하드웨어를 통한 실제 데이터 전송 |

---

## 3. 두 모델의 공통점과 차이점

### 공통점
- 계층적 레이어 구조를 가집니다.
- 하위 계층은 상위 계층에 서비스를 제공하고 상위 계층은 하위 계층을 사용합니다.
- 패킷 교환(Packet Switching) 방식을 기반으로 합니다.

### 차이점
- **세분화 정도**: OSI는 이론 중심의 7계층으로 매우 세밀하며, TCP/IP는 실제 상용 표준으로 통합된 구조입니다.
- **주도권**: 현재 실제 인터넷 통신은 **TCP/IP** 모델을 기준으로 동작합니다.

---

## 4. 왜 계층을 나누는가?
1. **표준화**: 서로 다른 제조사의 장비끼리도 통신이 가능하게 합니다.
2. **트러블슈팅 용이**: 특정 단계에서 문제가 발생하면 해당 계층만 확인하면 됩니다. (예: 인터넷이 안 되면 케이블(1계층)인지, IP 설정(3계층)인지 구분)
3. **독립성**: 한 계층의 기술이 발전하거나 수정되어도 다른 계층에 영향을 주지 않습니다.

# 쿠키(Cookie), 세션(Session), 그리고 캐시(Cache)

웹 개발을 하다 보면 이 세 단어를 항상 묶어서 듣게 됩니다. 셋 다 **"데이터를 어딘가에 저장해두고 재사용한다"**는 점은 같지만, **무엇을, 어디에, 왜** 저장하는지에 따라 큰 차이가 있습니다.

---

## 1. 한눈에 비교하기

| 구분 | 쿠키 (Cookie) | 세션 (Session) | 캐시 (Cache) |
| :--- | :--- | :--- | :--- |
| **저장 위치** | **사용자 브라우저** (로컬) | **웹 서버** | **브라우저 또는 서버 등** |
| **저장 목적** | 사용자 식별, 로그인 상태 유지 | 로그인 정보, 민감한 상태 유지 | **성능 향상** (로딩 속도) |
| **만료 시점** | 설정한 만료일 동안 유지 | 브라우저 종료 시 종료 (설정 가능) | 만료되거나 직접 삭제할 때까지 |
| **보안** | 낮음 (누구나 볼 수 있음) | 높음 (서버에 저장됨) | 관련 없음 |
| **용량** | 매우 작음 (4KB 이하) | 서버 사양에 따라 큼 | 용량 제한이 거의 없음 |

---

## 2. 쿠키 (Cookie)
- **개념**: 서버가 사용자의 브라우저에 남기는 **작은 데이터 파일**입니다.
- **특징**: 이름, 값, 만료일 등으로 구성되며, 사용자가 요청을 보낼 때마다 자동으로 서버에 전달됩니다.
- **예시**: "오늘 이 창 다시 보지 않기" 체크, 다크모드 설정 유지, 비로그인 상태에서 장바구니 담기.

## 3. 세션 (Session)
- **개념**: 사용자가 웹 사이트에 접속해서 브라우저를 닫을 때까지의 **상태를 서버에 저장**하는 방식입니다.
- **특징**: 서버는 '세션 ID'만 쿠키에 담아 사용자에게 주고, 실제 데이터는 서버의 메모리(또는 DB)에 보관합니다. 쿠키보다 보안이 우수합니다.
- **예시**: **로그인 정보 유지.** (로그인하면 서버가 "넌 누구야"라고 기억해두는 것)

## 4. 캐시 (Cache)
- **개념**: 이미지, CSS, JS 파일 처럼 용량이 크고 변하지 않는 **정적 리소스를 로컬에 복사**해두는 것입니다.
- **특징**: "로그인 했나?"가 궁금한 게 아니라, "아까 본 고화질 로고 이미지를 또 다운받기 싫어!"를 해결하기 위함입니다.
- **예시**: 사이트 재방문 시 로딩 속도가 빨라지는 현상.

---

## 5. 요약: "세트인가요?"

네, 넓은 의미에서는 **'웹 상태와 성능을 관리하는 3대장'** 세트라고 볼 수 있습니다!

1.  **쿠키 & 세션**: "이 사용자가 누구인지, 무엇을 하고 있는지"라는 **사용자 정보(State)**를 기억하기 위한 단짝입니다.
2.  **캐시**: 사용자가 누구인지는 상관없이 "어떻게 하면 정보를 더 빨리 보여줄까"라는 **성능(Performance)**을 위한 도구입니다.

---

### 💡 한 줄 구분
- **쿠키**: 기기(내 브라우저)에 저장하는 작은 정보 쪽지
- **세션**: 서버가 기억하는 나에 대한 정보
- **캐시**: 다시 쓰려고 미리 챙겨둔 짐 (이미지, 리소스 등)

# Simulation (시뮬레이션)

시뮬레이션 알고리즘은 **문제에서 제시한 상황을 코드로 그대로 구현하는 방식**입니다. 별다른 알고리즘적 기법(예: DP, Greedy 등)이 필요하기보다는 문제의 조건을 얼마나 빠짐없고 명확하게 코드로 옮기느냐가 핵심입니다.

## 1. 주요 특징
- **구현 중심**: 논리적 흐름을 코드로 변환하는 능력이 가장 중요합니다.
- **복잡한 조건**: 문제의 제약 사항이 많거나 단계가 여러 개인 경우가 많습니다.
- **시간 복잡도**: 보통 시뮬레이션 문제는 시간 제한이 넉넉한 편이지만, 구현량이 많아 실수하기 쉽습니다.

## 2. 필수 테크닉

### Direction Vectors (방향 벡터)
2차원 격자(Matrix)에서 이동할 때 자주 사용됩니다.
```python
# 상하좌우 이동을 위한 dx, dy
dx = [-1, 1, 0, 0]
dy = [0, 0, -1, 1]

# (x, y)에서 방향 i로 이동
nx = x + dx[i]
ny = y + dy[i]
```

### Constraints Check (범위 및 제약 조건 확인)
격자 밖으로 나가는지 또는 특정 장애물이 있는지 확인하는 과정이 필수입니다.
```python
if 0 <= nx < N and 0 <= ny < M:
    # 다음 동작 수행
```

### Matrix Rotation (배열 회전)
90도 회전 등 배열을 조작하는 기능이 자주 요구됩니다.
```python
def rotate_90(matrix):
    n = len(matrix)
    m = len(matrix[0])
    result = [[0] * n for _ in range(m)]
    for i in range(n):
        for j in range(m):
            result[j][n - 1 - i] = matrix[i][j]
    return result
```

## 3. 시뮬레이션 문제 풀이 전략
1. **문제 해석**: 요구사항(단계별 동작)을 메모장에 순서대로 정리합니다.
2. **함수화**: 각 단계를 함수로 분리하여 구현하면 디버깅이 훨씬 수월합니다.
3. **경계값 예외 처리**: 시작점, 끝점, 벽 등 특수 상황을 꼼꼼히 체크합니다.
4. **디버깅**: 중간중간 배열의 상태를 출력하여 로직이 맞는지 검증합니다.

- 로봇 청소기 이동 안내

# 🐳 Docker 개념 및 면접 대비 가이드

도커의 기본 개념, 작동 원리부터 기술 면접에서 자주 등장하는 질문들을 정리한 문서입니다.

---

## 1. 도커(Docker)란?
**도커**는 애플리케이션을 실행하는 데 필요한 모든 구성 요소(코드, 런타임, 시스템 도구, 라이브러리 등)를 **컨테이너**라는 독립된 환경에 패키징하여, 어느 환경에서나 일관되게 실행할 수 있게 돕는 오픈소스 플랫폼입니다.

- **핵심 키워드**: 가상화, 격리, 이식성, 일관성

---

## 2. 도커의 작동 원리 (핵심 기술)
도커는 하드웨어를 가상화하는 것이 아니라, **리눅스 커널의 기능을 활용하여 프로세스를 격리**하는 방식입니다.

### 1) 리눅스 커널 기술
- **Namespace (네임스페이스)**: 각 프로세스가 시스템 리소스를 독립적으로 보도록 격리하는 기술 (PID, Network, Mount 등).
- **Control Groups (cgroups)**: 각 컨테이너가 사용할 수 있는 하드웨어 리소스(CPU, 메모리 등)의 제한을 할당.

### 2) 계층형 파일 시스템 (Union File System)
- 도커 이미지는 여러 개의 **레이어(Layer)**로 구성됩니다.
- 이미지를 빌드할 때 변경된 부분만 새로운 레이어로 쌓으므로 효율적인 저장과 전송이 가능합니다.

---

## 3. 컨테이너 vs 가상 머신(VM)
가장 많이 나오는 비교 질문입니다.

| 비교 항목 | 컨테이너 (Docker) | 가상 머신 (VM) |
| :--- | :--- | :--- |
| **핵심 기술** | OS 레벨 가상화 (커널 공유) | 하드웨어 가상화 (Hypervisor 위) |
| **운영체제** | 호스트 OS 커널을 공유 | 각 VM마다 Guest OS 설치 |
| **속도** | 초 단위로 매우 빠름 | 분 단위 (OS 부팅 필요) |
| **리소스 사용** | 가볍고 효율적 (필요한 만큼만 사용) | 무겁고 중복됨 (Guest OS 차지 비중) |

---

## 4. 도커 기술 면접 예상 질문 & 답변 팁

### Q1. 도커 이미지(Image)와 컨테이너(Container)의 차이는?
- **답변**: **이미지**는 애플리케이션 실행에 필요한 모든 파일과 설정이 담긴 '실행 파일(정적 상태)'이고, **컨테이너**는 이 이미지를 바탕으로 실제 메모리에 올려 실행한 '인스턴스(동적 상태)'입니다. 붕어빵 틀(이미지)과 붕어빵(컨테이너)의 관계로 비유할 수 있습니다.

### Q2. 도커를 사용했을 때 얻는 가장 큰 이점은?
- **답변**: "내 로컬에서는 되는데 서버에선 안 돼요"라는 문제를 해결해줍니다. 환경의 일관성을 제공하여 개발-테스트-운영 서버 어디서든 똑같은 환경을 즉시 구축할 수 있고, 서비스 확장이 유연합니다.

### Q3. 도커 컴포즈(Docker Compose)를 왜 사용하나요?
- **답변**: 여러 개의 컨테이너(예: Spring Boot 앱, MySQL, Redis 등)로 구성된 애플리케이션을 하나의 설정 파일(`yaml`)을 통해 일괄 관리하기 위함입니다. 컨테이너 간의 네트워크 연결과 실행 순서를 한 번에 제어할 수 있습니다.

### Q4. 도커 이미지 레이어(Layer) 구조란 무엇인가요?
- **답변**: Dockerfile의 각 명령어(RUN, COPY 등)가 실행될 때마다 하나의 레이어가 형성됩니다. 읽기 전용 레이어들이 층층이 쌓여 하나의 이미지가 되며, 컨테이너를 실행하면 그 위에 '쓰기 가능한 레이어'가 한 층 추가됩니다. 이미 사용된 레이어는 캐싱되어 재사용되므로 빌드 속도가 매우 빠릅니다.

### Q5. 컨테이너가 삭제되면 그 안의 데이터는 어떻게 되나요?
- **답변**: 기본적으로 컨테이너 내부 데이터는 휘발성이라 컨테이너 삭제 시 함께 사라집니다. 이를 영구적으로 보존하기 위해 **도커 볼륨(Volume)**이나 바인드 마운트(Bind Mount)를 사용하여 호스트 OS의 경로와 연결해야 합니다.
